# 起步

开始对代码的结构走一个简单的调整。api 不应该直接放在根目录下。 在根目录下新建一个 app 文件夹，将 api 文件夹放在这个 app 文件加下。

app.js的代码也进行相应的修改。

requireDirectory(module, './api/api', { visit: whenLoadModule })
1

接下来我们对 app.js 进行小小的重构，那么为什么要进行重构呢?

app.js不应写过多的代码，虽然目前就写requiredirectory，好像没有什么影响 但是随着项目的进行，如果所有的代码都写在app.js的话，会让这个入口文件看起来非常的臃肿。

# 分离 requiredirectory

那么我们怎么来分离 requiredirectory 这类的代码呢？

在根目录下新建一个文件夹，名字叫做 core，后续一些公共的方法或者是类都可以写在这个文件下。我们在 core 文件夹下新建一个 init.js 文件。我们需要用类的思想来组织 requiredirectory 这类的代码。

我们在init.js里定义一个类initManager，在initManager中定义一个静态的方法 initLoadRouters用来加载全部的路由。其中的方法逻辑与上一节在app.js写的逻辑是类似。

requireDirectory(module, './api', { visit: whenLoadModule })

function whenLoadModule(obj) {
  if (obj instanceof Router) {
    app.use(obj.routes())
  }
}

# process.cwd()

按 f5 运行 发现报错了

报错信息显示没有找到文件,想想一般都是文件路径问题。我们把代码剪切过来的时候文件路径已然发生了变化。这样就可以了！！


 








static initLoadRouters() {
    const apiDirectory = '../app/api';
    requireDirectory(module, apiDirectory, { visit: whenLoadModule });
    function whenLoadModule(obj) {
      if (obj instanceof Router) {
        InitManager.app.use(obj.routes());
      }
    }
  }

其实把api的路劲直接写在代码里是不合适的，这样叫做硬编码 。 因为这样前面我们调整了代码位置就报错了,显然这是不合适的。

下面有两种解决方法：

  • 我们可以把路径写在配置文件里面path config 然后在配置文件里读取。
  • 我们可以使用绝对路径，因为 api 的路径相对于根目录是不会有变化了。

我们使用第二种方法，那么如何找到绝对路径的？

在 node.js 里有一个全局变量 process 的 cwd 方法 process.cwd() 我们断点调试一下,在process.cwd()那一行打一个断点按F5 然后选中process.cwd()右击鼠标点击调试求值就可以看到 命令行就会输出绝对路径,这个绝对路径就是项目文件夹的路径。

知道了这个用法，initLoadRouters我们就可以这样写了。

static initLoadRouters() {
   const apiDirectory = `${process.cwd()}/app/api`;
   requireDirectory(module, apiDirectory, { visit: whenLoadModule });
   function whenLoadModule(obj) {
     if (obj instanceof Router) {
       InitManager.app.use(obj.routes());
      }
    }
}

# 起步

Koa导入项目

const Koa = require('Koa')
1

使用KOA 必须实例化koa

const app = new Koa()
1

实例化koa => app我们称为应用程序对象，其中可以包含了许多的中间件

我们有了应用程序对象(app)后，即可调用其上面的方法listen以启动整个koa框架

listen可传入许多参数,目前先传入端口号port 3000

提醒

注意：所输入的端口号不能被其他的应用程序占用的，否则报错

运行代码 node app.js,我们可以看到命令行一直在闪烁。这说明应用程序处于阻塞状态，只有应用程序处于阻塞状态，才能监听到从前端发过来的HTTP请求。

# 利用浏览器发送HTTP请求

我们如何接收前端的HTTP请求 则需要中间件（实际上就是一个函数）?

一般前端发送请求都是来服务端获取数据，可以理解为请求触发服务端的方法来使得服务端发送数据给前端，这里写一个简化版的 即请求触发了服务端的方法（省略了服务端发送这一环节）重点在于前端发送的HTTP如何调用服务端的方法。

function test(){
    console.log('hello,lemon')
}
1
2
3

既然需要调用test（）方法 我直接在app.js中调用不就好了吗？非也非也

如果调用方法这一步直接写在app.js就与我们最初的目的背道而驰了，我们最近的目的是服务端接收到了前端的HTTP请求后再调用相应的方法

那么如何来实现我们的目的呢，这里我就需要使用中间件的概念。

一个函数如何成为一个真正的中间件呢?

我们定义了函数之后，将其注册到应用程序对象中。应用程序对象提供了use方法 use方法可以接收函数 使用app.use就可以把函数注册到应用程序对象中去，这样被注册函数就可成为一个中间件

例如:

function test(){
    console.log('hello,1lemon')
}
app.use(test)

这时，我们再启动应用程序 node app.js。

我们发现**test()并没有被执行，这一点和直接在app.js调用test()**是不同的

那么如何才能调用这个test()函数呢，这就是我们刚刚说的最初的目的，要接收到从前端发来的http请求后才能触发**test()**函数。

从前端发送http请求有很多种方法，其中最简单的方式就是从浏览器中发送http请求，因为我们是在本地搭建的koa服务器,所以可以使用localhost或者127.0.0.1加上指定的端口号port这就是我们服务器的地址,在输入服务器地址后 显示:

这是没有关系的，因为服务器没有返回任何的信息所以显示为Not Found，但是我们可以发现控制台出现了hello,1lemon。

编译器中打印了东西，说明浏览器发送的http请求调用了服务端的**test()**函数

通常情况下，会直接在app.use()中使用匿名函数 可直接使用箭头函数

function test(){
    console.log('hello,1lemon')
}
app.use(test)
---
app.use(() =>{
    console.log('hello,1lemon')
})

# 利用next调用下一个中间件

这里提出一个问题: 一个应用程序上只能有一个中间件吗？

并不是，一个应用程序可以注册多个中间件

来试一试！

app.use(() =>{
    console.log('hello,1lemon')
})

app.use(() =>{
    console.log('hello,2lemon')
})
1
2
3
4
5
6
7

更改完代码记得重启node服务器。

之后我们再去浏览器访问127.0.0.1:3000，再发送一遍http请求，这时候我们发现只有第一个中间件生效了，多次刷新浏览器，其结果是一致的，只打印了第一个中间件的内容hello,1lemon。

这是为什么呢？

我们来看这两个中间件，可以发现两个中间件注册是有顺序了。在koa中只能自动帮你执行第一个中间件，所以解释了为什么只打印了第一个中间件的内容。后续的中间件都是需要开发者自己进行调用。

那么如何调用呢？

koa在每次调用中间件的时候都会给中间件传递两个参数 这两个参数是由koa内部的机制帮我们传入的，不需要我们开发者去管理。第一个ctx通常叫做上下文，next就是下一个中间件函数。按照惯例，每一个中间件都需要传入这两个参数。 我们来实验一下

app.use((ctx,next) =>{
    console.log('hello,1lemon')
    next()
})

app.use((ctx,next) =>{
    console.log('hello,2lemon')
})

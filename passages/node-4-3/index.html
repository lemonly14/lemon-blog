# 起步

上一节我们提到 **try…catch…**来捕获异常，不过只对于同步的方法有效果的， 如果方法内部包含了一些异步的操作，**try…catch…**有时候是捕获不到异常的。 我们举个例子，在 func3 中使用 setTimeout 来模拟异步操作抛出一个异常。然后在 func2 中调用 func3 并使用 **try…catch…**尝试一下能不能捕捉到 func3 的异常。 记住在代码最后调用func1，让整个函数调用链开始执行。

function func1() {
  func2()
}

async function func2() {
  try {
    await func3()
  } catch (error) {
    console.log('error')
  }
}

async function func3() {
  setTimeout(() => {
    throw new Error('error')
  }, 1000)
}

# 利用 async 和 await 处理异常

在异步编程模型里面异常处理是比较难的，特别是以前没有Promise async 和 await,只能靠回调函数来处理异常，这是非常麻烦的。现在有了 Promise async 和 await 相对而言容易很多。容易很多仅对写法而言，理解原理的话还是有一定难度的。（异步编程模型的异常处理链条。）

提示：Promise async 和 await 是不分家的，使用 async 和 await 就必须要有 Promise 的支持。

# 结论性内容

以后在进行 koa 编写的时候，如果某一个函数返回的是 Promise 的话，其他函数调用该函数的时候就可以使用 async 和 await 来简化异常链条。之后使用**try…catch…**就可以捕获到异常了。

需要强调的有两点：

  • 前提条件是函数返回的是一个 Promise，才可以使用 async 和 await 加 **try…catch…**进行处理。
  • 在函数调用链里面，某一个函数使用了 async 和 await，建议在函数调用链中的函数都加上 async 和 await。否则还是会出现 promise 没有处理异常。

贴上代码:

function func1() {
  func2()
}

async function func2() {
  try {
    await func3()
  } catch (error) {
    console.log('error')
  }
}

async function func3() {
  return new Promise((resolve, rejects) => {
    setTimeout(() => {
      const r = Math.random()
      if (r < 0.5) {
        rejects('Error')
      }
    })
  })

  setTimeout(() => {
    throw new Error('error')
  }, 1000)
}

为什么async 和 await 可以捕获到Promise 的异常？

关键在于，await 可以对表达式求值，如果表达式是一个 Promise，那么它就会等待 Promise，上述的 Promise 有可能是正确，也可能是错误。如果是错误，Promise 就会抛出错误，后续 await 就可以捕获到这个错误异常。

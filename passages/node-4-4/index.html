# 起步

本节开始 再次强调上一节的一个内容，函数调用链中出现了异步操作必须加上 await！

我们知道 func3 不是 throw 一个异常。可是 func2 能够**try…catch…**到 func3 的异常，这是因为 func2 的 await 释放了这个异常。在一个较长调用链里面，某一处不加 await 的话，Promise 可能会产生一个没有处理的异常。

我们来复现一下这种情况，删除 func2 里面的 await。

function func1() {
  func2()
}

async function func2() {
  try {
    func3()
  } catch (error) {
    console.log('error')
  }
}

async function func3() {
  return new Promise((resolve, rejects) => {
    setTimeout(() => {
      const r = Math.random()
      if (r < 0.5) {
        rejects('Error')
      }
    })
  })

  setTimeout(() => {
    throw new Error('error')
  }, 1000)
}

我们可以看到Unhandled promise rejection.这些字眼。它们说明 Promise 产生了异常，外部却没有处理这些异常。

之前我们说过不管同步异步都使用try…catch…，这很麻烦！所以我们需要一个全局的异常处理，可以监听到任何异常。

# 实现全局监听

那么如何实现这种机制呢？

使用中间件实现这种机制。在项目的根目录下，新建一个文件夹 middlewares。

假如说，classic api 中抛出了一个异常，我们需要在全局异常处理里面监听到这个错误，并且输出一段提示信息告诉用户为什么错误。










 




const Router = require('koa-router')
const router = new Router()
router.post('/v1/:id/classic/latest', (ctx, next) => {
  const path = ctx.params
  const query = ctx.request.query
  const header = ctx.request.header
  const body = ctx.request.body
  ctx.body = { key: 'classic' }

  throw new Error('API Exception')
})

module.exports = router

const catchError = async (ctx, next) => {
  try {
    await next()
  } catch (error) {
    ctx.body = '服务器走丢了...'
  }
}

module.exports = catchError

现在我们尝试把编写的中间件注册到app中。






 

 





const Koa = require('Koa')
const app = new Koa()
const InitManager = require('./core/init')
const parser = require('koa-bodyparser')

const catchError = require('./middlewares/exception')
app.use(parser())
app.use(catchError)
process.cwd()
InitManager.initCore(app)

app.listen(5000)

# 全局异常处理的好处

以后编写业务代码的时候，无论函数调用链有多长，都不需要try…catch…，因为在整个函数调用链的最外层做了一个中间件形式的全局异常处理函数，它可以监听到整个函数调用链上任何一个异常，并且集中把所有的错误异常处理的逻辑都写在catchError里面的**try…catch…**中。

全局异常处理应用到编程中一个重要的思想 AOP 面向切面编程

# 获取 http 动词和路径

这里我们可以看到它的 http 动词是 get，路径是/classic/latest。

那么我们如何在中间件中获取到 get 和 /classic/latest 呢？

我们知道中间件自带的两个参数 ctx 和 next, next()代表着下一个中间件 所以我们要在ctx 上下文上获取。

我们可以先进入 koa 的官网找到request.method和request.path。两者分别来获取 http 动词和路径 但是真正调用的时候使用的是ctx.method和ctx.path这是 koa 内部代理的原理。 详情自行参阅koa 文档

ctx.method和ctx.path 与 request.method和request.path 是等效的

我们来尝试一下：

app.use(async (ctx, next) => {
  console.log(ctx.path)
  console.log(ctx.method)
})

# ctx.body

在 koa 里面，如果你想把最终结果返回到客户端，是不能把内容写在中间件里面的 return。 我们必须把内容保存到 ctx.body 上面，koa 会读取 ctx.body 的内容并最终返回到客户端上面。

app.use(async (ctx, next) => {
  console.log(ctx.path)
  console.log(ctx.method)
  if (ctx.path === '/classic/latest' && ctx.method === 'GET') {
    ctx.body = 'classic'
  }
})

我们看到了返回了字符串，但是在实际的编程中是不会直接返回字符串，一般是返回一个 json 那么如何返回 json 呢?

我们直接把一个JS 对象赋值给ctx.body

app.use(async (ctx, next) => {
  console.log(ctx.path)
  console.log(ctx.method)
  if (ctx.path === '/classic/latest' && ctx.method === 'GET') {
    ctx.body = { key: 'classic' }
  }
})

# api 路由

我们为什么在中间件要写 if 判断呢呢？

这是因为我们要处理的api非常多，如果不判断的话，是无法确定用户访问的是哪个接口，所以要先做判断再做相应的处理。

可是如果项目有 100 个 api，那么我们要写 100 个 if 判断吗？（灵魂拷问）

显然这是不现实的，这么写的话代码不好看，不好读，不好维护。这就引出了我们路由系统的概念。基本上 koa 路由系统都是基于一个第三方的路由系统的我们直接引入就好了

我们可以在 package.json 里面可以看到：

我们已经安装了koa-router，直接引入

const router = require('koa-router')
1

koa-router 详细内容可以自行参阅。

这里演示了 koa-router 的最基本用法

大致有三步

  • 实例化 koa-router
  • 编写一系列的路由函数
  • 调用 router.routes()把中间件注册到 app 应用程序上

第一步：（实例化 koa-router）

const Router = require('koa-router')
const router = new Router()
1
2

第二步：（然后用编写一系列的路由函数）

router.get('classic/latest', (ctx, next) => {
  ctx.body = { key: 'classic' }
})
1
2
3

小贴士

get 后面带有两个参数，一个是路径，一个是中间件 （router.post,router.put,router.delete）同理使用 这四个分别对应着 查询 新增 更新 删除

第三步：(注册到 app 应用程序)

app.use(router.routes())

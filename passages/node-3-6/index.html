# 获取所有路由模块

关键在于如何能获取到路由文件中的所有路由模块。

其实在之前我们就在package.json文件中写好了，引入了'require-directory'。

那我们应该如何使用 require-directory 文档呢？ require-directory 文档

首先，我们在app.js里面导入require-directory。



 



 





const Koa = require('Koa')
const app = new Koa()
const requireDirectory = require('require-directory')
const classic = require('./api/v1/classic')
const book = require('./api/v1/book')

const modules = requireDirectory(module, './api/v1')
app.use(classic.routes())
app.use(book.routes())

app.listen(5000)

# 注册路由

接下来，我们要把所有拿到的路由注册到 app应用程序上。我们考虑一下如何实现这一步呢？

我们使用一个 *for..in..*循环遍历将所有的路由模块都注册到 app 上。这里都有一个问题了如果我们不做判断将所有的模块都注册到 app 上，可能是会报错的，不一定所有的模块都是路由模块。所以再加上一个是否为路由模块的判断

const Koa = require('Koa')
const app = new Koa()
const requireDirectory = require('require-directory')
const classic = require('./api/v1/classic')
const book = require('./api/v1/book')

const modules = requireDirectory(module, './api')
for (let r in modules) {
  if (r instanceof Router) {
    app.use(r.routes())
  }
}

app.listen(5000)

其实我们这样遍历寻找 router 模块不是很好。有一个更好的办法，我们继续深挖 requiredirectory 这个模块的高级方法。

在高级方法里我们还可接收第三个参数，第三个参数是一个对象里面可以写一个属性 visit 可以被赋值一个函数，我们自定义一个函数 whenLoadModule 每当 requiredirectory 加载到一个模块的时候都会调用这个函数，这个函数实际上就是一个回调函数，执行这个回调函数前，都会把获取得到的 module 传递到这个回调函数，我们用一个参数 obj 接住。在代码块里做出相应的判断。判断是否是一个 router ？如果是就进行注册。

const Koa = require('Koa')
const app = new Koa()
const requireDirectory = require('require-directory')

requireDirectory(module, './api', { visit: whenLoadModule })

function whenLoadModule(obj) {
  if (obj instanceof Router) {
    app.use(obj.routes())
  }
}

app.listen(5000)

总结：使用了requiredirectory之后我们就可以自动注册路由模块了，不需要像之前一样手动注册，会方便很多。
好的代码是可以提高效率的，如果没有自动注册路由我们必然会浪费很多的时间。所以我们要动脑筋思考有没有办
法简化写代码的过程而不是为了实现功能而实现功能，只能思考才能提高水平
